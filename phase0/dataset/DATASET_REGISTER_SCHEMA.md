# Схема реестра датасета V0.1

## Обязательные поля

| Поле | Смысл |
|---|---|
| `sample_id` | уникальный ID файла, например `IMG-000001` |
| `family_id` | общий ID исходника и всех его производных |
| `parent_sample_id` | ID непосредственного исходника; пусто для оригинала |
| `split` | `development`, `internal_holdout` или `external_holdout` |
| `ground_truth` | `real` или `ai_generated` |
| `truth_basis` | почему происхождение известно |
| `source_type` | камера, генератор, открытый датасет, контролируемое преобразование |
| `source_name` | модель камеры, название генератора или датасета |
| `source_version` | версия модели/сервиса, если известна |
| `original_filename` | исходное имя файла |
| `stored_filename` | имя в наборе |
| `file_format` | JPEG, PNG, WebP и т.п. |
| `width`, `height` | размеры в пикселях |
| `sha256` | SHA-256 сохранённого файла |
| `acquired_at` | дата получения |
| `acquired_by` | кто получил файл |
| `license_or_permission` | основание законного использования |
| `transformation` | `none`, JPEG, resize, crop, screenshot и т.п. |
| `transformation_params` | точные параметры преобразования |
| `notes` | ограничения и важные замечания |

## Поля результата

Результаты детекторов не записываются в ground truth. Для них используются отдельные поля или отдельный журнал:

- версия инструмента и SHA веса;
- дата запуска;
- `community_score`;
- `spai_score`;
- ошибка/отказ;
- время выполнения;
- техническая среда.
