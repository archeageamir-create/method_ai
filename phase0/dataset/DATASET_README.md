# Датасет проекта

Этот раздел хранит только проверочные материалы, происхождение которых известно заранее. Материалы реальных дел сюда не загружаются.

## Части набора

- `02_Development` — открытая рабочая часть; по ней можно настраивать код и находить ошибки.
- `03_Internal_holdout` — закрытая внутренняя часть; не используется при настройке.
- `04_External_holdout` — материалы из независимого источника для финальной проверки переносимости.

## Простое правило

Каждое изображение получает:

1. уникальный `sample_id`;
2. известное происхождение `real` или `ai_generated`;
3. SHA-256;
4. источник и условия получения;
5. назначенную часть набора;
6. связь с исходником через `family_id`, если это JPEG-копия, crop, resize или другой производный файл.

Одинаковые или родственные изображения нельзя распределять между development и holdout: иначе тест будет искусственно завышен.

## Стартовый объём

Для первого dry run достаточно 20 реальных и 20 AI-изображений. Это проверит процесс, но не точность методики. После исправления процесса набор расширяется по плану валидации.
