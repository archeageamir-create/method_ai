# Источник правды и синхронизация с Google Drive

**Дата решения:** 14 июля 2026 года.  
**Владелец:** Смагулов Амирхан Каиржанович.

## Основное правило

Источник правды проекта — локальный Git-репозиторий:

`C:\Users\arche\Documents\Методика по ИИ`

GitHub-репозиторий `archeageamir-create/method_ai` является основным удалённым хранилищем коммитов. Google Drive не является местом первичного редактирования: он используется как дополнительное зеркало и как удобная копия документов для просмотра.

## Порядок работы

1. Документ или скрипт изменяется локально.
2. Изменение проверяется.
3. Изменение фиксируется отдельным Git-коммитом.
4. Коммит отправляется в `https://github.com/archeageamir-create/method_ai`.
5. Скрипт синхронизации при необходимости копирует точное содержимое коммита в управляемую папку Google Drive.
6. В облачном зеркале сохраняется `SYNC_METADATA.json` с hash коммита и временем синхронизации.
7. В `PROJECT_HISTORY.bundle` сохраняется полная Git-история для дополнительного восстановления репозитория на другом компьютере.

Незакоммиченные изменения не синхронизируются. Если рабочее дерево грязное, скрипт останавливается, чтобы облако не получило промежуточную редакцию.

## Облачное зеркало

Локальная точка Google Drive Desktop:

`G:\Мой диск\ИИ методика\_LOCAL_REPO_MIRROR`

Папка `_LOCAL_REPO_MIRROR` управляется только скриптом `tools/sync-to-cloud.ps1`. Ручное редактирование файлов внутри неё не меняет источник правды и будет перезаписано следующей синхронизацией.

Существующая тематическая структура Google Drive сохраняется. Зеркало создаётся отдельно и не удаляет файлы вне `_LOCAL_REPO_MIRROR`.

Текущее содержание основной Google-методики сохранено локально в `methodology/MAIN_METHOD_CURRENT.md`, а текст презентации — в `methodology/PRESENTATION_CURRENT_OUTLINE.md`. Текстовые локальные копии являются контролируемым содержанием. Google Doc и Google Slides пока сохраняются как публикационные копии, нужные для оформления и показа.

## Что не попадает в зеркало

В зеркало попадают только файлы последнего Git-коммита. Поэтому автоматически исключаются:

- личные HEIC/JPEG и другие локальные датасеты из `work/datasets/`;
- веса моделей и скачанные репозитории детекторов;
- виртуальные окружения Python;
- временные файлы, кэши и рабочие outputs;
- любые материалы дела, пока они явно не включены в контролируемый репозиторий по отдельному решению.
- крупные сторонние PDF, DOCX и PPTX, которые нельзя автоматически публиковать в открытом GitHub без отдельного основания.

## Запуск

Из корня проекта:

```powershell
& .\tools\sync-to-cloud.ps1
```

Скрипт проверяет чистоту Git, создаёт архив `HEAD`, разворачивает его во временную папку и зеркалирует только эту папку в `_LOCAL_REPO_MIRROR`. Удаление устаревших файлов возможно только внутри специально проверенного пути зеркала.

Для восстановления полной истории на другом компьютере:

```powershell
git clone PROJECT_HISTORY.bundle "Методика по ИИ"
```
